# CLAUDE.md — BAZA SVL Landing

## Проект

Одностраничный лендинг для деревообрабатывающей производственной базы **BAZA SVL**.  
Стек: HTML + CSS + Vanilla JS. Никаких фреймворков, никакого билда — открывается напрямую в браузере.

**Развёрнут на GitHub Pages:** https://profikate.github.io/baza-svl/  
Репозиторий: https://github.com/ProfiKate/baza-svl

Полное описание проекта — в [project.md](project.md). ТЗ — в [ТЗ.pdf](ТЗ.pdf).

## Файлы, которые редактируются

| Файл | Назначение |
|------|-----------|
| `index.html` | Вся разметка. Один файл, все секции внутри |
| `css/style.css` | Все стили. CSS-переменные в `:root` вверху файла |
| `js/script.js` | Вся логика: скролл, бургер, AOS, форма |

Логотип: `Лого белый текст.png` (хедер + футер).  
OG-изображение для мессенджеров: `og-image.png` (копия `Лого вертикаль.png` с ASCII-именем).  
Медиафайлы (PNG, JPG, MP4) лежат в корне — не переносить и не переименовывать.  
Новые файлы не создавать без явной просьбы пользователя.

## Дизайн-система

Все цвета — через CSS-переменные в `style.css`, блок `:root`. Не использовать хардкод цветов в HTML или inline-стилях.

Ключевые переменные:
- `--amber` / `--amber-hover` — акцент, CTA-кнопки, иконки
- `--oak` / `--oak-bg` — тёплый деревянный акцент
- `--graphite` / `--graphite-2` / `--graphite-3` — тёмные фоны

Шрифты: **Montserrat** (заголовки, кнопки) и **Manrope** (текст). Оба подключены через Google Fonts в `<head>`.

## Анимации

Scroll-анимации через **AOS.js 2.3.4** (CDN). Атрибуты на элементах: `data-aos="fade-up"`, `data-aos-delay="100"` и т.д.  
Настройки AOS в `script.js` — `once: true`, `duration: 600`.

## Адаптивность

Брейкпоинты в конце `style.css`:
- `≤ 1024px` — перестройка сеток услуг и продукции
- `≤ 768px` — бургер-меню, все сетки в 1 колонку
- `≤ 480px` — кнопки и мессенджеры на всю ширину

## Форма заявки

Форма (`#contactForm`) работает без бэкенда.  
Кнопка — `type="button"` с `onclick="handleFormSubmit()"` (глобальная функция в `script.js`).  
После успешной отправки `form.innerHTML` заменяется success-блоком — страница не скроллит.  
Внутри success-блока есть ссылка «Написать в Telegram» с предзаполненным текстом заявки.

## Мобильный хедер и бургер-меню

На `≤768px` `.nav` переключается в `position: absolute; top: 100%` (относительно хедера, не viewport).  
В закрытом состоянии: `transform: translateY(-200%); pointer-events: none`.  
В открытом (`.nav.open`): `transform: translateY(0); pointer-events: auto`.

Использовать именно `position: absolute`, а не `position: fixed` — иначе iOS Safari баг (fixed внутри fixed с backdrop-filter).

В `script.js` код бургера идёт **первым** (до AOS.init), AOS обёрнут в `try/catch` — чтобы бургер работал даже если CDN не загрузился.

## Правила работы с кодом

- Комментарии в коде не добавлять — секции в HTML уже размечены блоками `<!-- === НАЗВАНИЕ === -->`
- Не добавлять новые зависимости без необходимости
- Не трогать имена медиафайлов — пути в HTML жёстко привязаны к именам на кириллице
- Google Maps в секции `#contacts` — реальные координаты базы: `53.4205527, 27.0466418`, спутниковый вид (`&t=k` в URL embed). Hybrid с подписями — `&t=h`.
- Секция `#contacts`: три равные колонки в одном ряду — контакты | форма | карта (карта крайняя справа). На ≤1024px — стопкой. Нет обёртки `.contacts__right` — все три элемента прямые дочерние `contacts__inner`.
